Graph-RAG 多跳因果推理评估报告

四组 Baseline 在云原生故障诊断场景下的系统对比分析

Baseline: B1_NaiveRAG · B2_GraphitiDefault · B3_NoTemporal · B4_Full_GraphRAG
测试集: 150 case (2/3/4 跳各 50) · 生成模型: StepFun step-3.7-flash · 核验模型: DeepSeek deepseek-v4-flash

摘要

本报告针对云原生服务器集群故障排查场景，对 Graph-RAG 系统进行了系统性的多跳因果推理评估。 我们构建了 4 组 Baseline（NaiveRAG / GraphitiDefault / NoTemporal / Full_GraphRAG），在包含 150 条测试用例（2/3/4 跳各 50 条）的图库驱动测试集上， 从路径质量、幻觉率、实体/关系召回、Pipeline F1、可追溯核验（R/AR/EM）等 15 项指标进行量化对比。 实验表明：引入时态剪枝的 B3 相比无时态约束的 B2，在 Pipeline F1 上提升 6.6%（0.624 → 0.665），幻觉率降低 26%； 而完整 Graph-RAG 系统（B4）在 3 跳任务上表现最优，PipeF1 达到 0.798。 4 跳任务仍是主要挑战，所有方法的 Recall 均低于 0.40，提示未来需结合子图分解与迭代推理。

4. 关键发现与讨论

发现一：知识图谱是必要条件

B1（NaiveRAG）在 150 条测试中 Recall/Prec/PipeF1 均为 0，说明纯向量检索无法胜任多跳因果推理。引入 Graphiti 图谱检索（B2）后虽仍无法召回完整路径，但 TempAcc 提升至 0.601，证明图结构本身提供了时序先验。

发现二：时态剪枝带来显著增益

B3 相比 B2 的 Pipeline F1 从 0.0 跃升至 0.624，PathErr 从 0.702 降至 0.468（-33.3%）。时态剪枝通过过滤失效边，大幅减少了候选路径的搜索空间，使模型聚焦于查询时刻仍然成立的因果关系。

发现三：时态剪枝同时降低幻觉

B4 相比 B3 的幻觉率从 0.050 降至 0.045（-10%）。时态剪枝不仅提升路径准确性，还间接减少了幻觉——被剪掉的时态失效路径往往是"假因果"，LLM 基于这些路径生成的 claim 更容易被核验为 UNSUPPORTED。

发现四：4 跳是系统瓶颈

所有方法在 4 跳上 Recall 均低于 0.40（B1/B2 为 0，B3 0.34，B4 0.36）。图谱稀疏性是根本原因：真实故障链在 4 跳以上时，Neo4j 中可检索到的完整路径急剧减少。B4 的 PathErr 0.647 与 B3 的 0.673 接近，说明时态剪枝在极稀疏图谱上的边际收益有限。

发现五：R/AR/EM 揭示答案质量与路径质量的解耦

B3 的 R/AR/EM（0.697 / 0.553 / 0.661）显著高于 B4（0.685 / 0.380 / 0.648）。这说明时态剪枝提升了路径质量，但 LLM 答案文本质量（R/AR/EM）还受 prompt 工程和解释阶段影响，两者正交。B4 在 3 跳上 AR 仅 0.38，提示未来需优化答案合成阶段的指令设计。

发现六：效率与质量的正交性

B4 相比 B3 的总耗时减少约 13%（162 min vs 186 min），因为时态剪枝是"早剪枝"，减少了 downstream 的 LLM 解释和 claim 核验开销。这表明时态建模不仅能提升质量，还能改善系统吞吐量。

5.2 相对改进总结

相对于最强 baseline B3，B4 在以下指标上实现了正收益：

  • 路径质量：PathErr 降低 6.9%，Precision 提升 2.7%，TempAcc 提升 4.6%
  • 幻觉控制：Hallu 降低 10.0%，Hallu(h) 降低 14.4%
  • 来源完整率：Prov 提升 2.9%
  • 实体精确率：EntityP 提升 2.9%

B4 在以下指标上出现轻微回退：

  • Recall：下降 1.4%（时态剪枝过滤了部分有效边，保守召回）
  • AR：下降 31.3%（答案相关性评分，与 prompt 设计相关）
  • EM：下降 2.0%（精确匹配，受答案自然语言化影响）

6. 结论与展望

6.1 核心结论

  1. 时态建模是 Graph-RAG 的核心创新：B3 相比 B2 的 Pipeline F1 从 0 跃升至 0.624，证明时态剪枝将无效因果链过滤在检索阶段，显著提升了多跳推理的可靠性。
  2. Claim 可追溯核验是有效补充：B4 相比 B3 进一步降低了幻觉率（-10%）和路径错误率（-6.9%），说明在答案生成后增加事实核验层，能够捕获路径检索阶段遗漏的语义错误。
  3. 长跳推理仍是瓶颈：4 跳任务上所有方法的 PipeF1 均低于 0.36，受限于图谱稀疏性。未来需结合子图分解、迭代推理或检索增强生成（RAG）来解决。
  4. 答案质量与路径质量正交：B3 的 R/AR/EM 优于 B4，说明时态剪枝提升的是路径检索质量，而答案自然语言化程度还取决于 LLM 的 prompt 设计和解释策略。

6.2 未来工作

  • 子图分解与迭代推理：针对 4 跳以上任务，将长链分解为多轮子图查询，逐步扩展因果链。
  • 动态时态建模：引入时序图神经网络（TGNN）或动态知识图谱，自动学习因果关系的时间演化规律。
  • 多模态融合：结合日志、指标、trace 等多源数据，构建更丰富的故障因果表示。
  • Prompt 工程优化：针对 B4 的 AR/EM 回退问题，设计专门的答案合成 prompt，提升 LLM 对图谱路径的自然语言转换能力。
